Web Accessibility (a11y)

Making Web Content Accessible

Accessibility ensures web content is usable by people with disabilities:


<!-- Proper Heading Structure -->
<h1>Main Page Title</h1>
<h2>Section Title</h2>
<h3>Subsection Title</h3>

<!-- Image Accessibility -->
<img src="logo.png" alt="Company Logo" aria-label="Company Logo">
<img src="decorative.png" alt="" role="presentation">

<!-- Form Accessibility -->
<form>
    <label for="name">Name:</label>
    <input 
        type="text" 
        id="name" 
        name="name" 
        aria-required="true"
        aria-describedby="name-help"
    >
    <span id="name-help">Enter your full name</span>
</form>

<!-- ARIA Landmarks -->
<header role="banner">
    <nav role="navigation" aria-label="Main navigation">
        <!-- Navigation items -->
    </nav>
</header>

<main role="main">
    <article role="article">
        <!-- Content -->
    </article>
    <aside role="complementary">
        <!-- Sidebar content -->
    </aside>
</main>

<!-- Skip Navigation -->
<a href="#main-content" class="skip-link">
    Skip to main content
</a>

<!-- Accessible Tables -->
<table role="grid">
    <caption>Monthly Sales Data</caption>
    <th scope="col">Month</th>
    <th scope="col">Sales</th>
</table>

Accessibility Features:

  • ARIA roles - Define element purposes
  • ARIA labels - Provide text alternatives
  • Alt text - Describe images
  • Form labels - Associate labels with inputs
  • Semantic structure - Use proper heading hierarchy
  • Skip links - Allow keyboard navigation
  • Table headers - Properly scope table headers

Practice Exercise

Create an accessible webpage that includes:

  • Proper heading structure
  • Accessible images with alt text
  • A form with proper labels and ARIA attributes
  • ARIA landmarks for major sections
  • A skip navigation link
