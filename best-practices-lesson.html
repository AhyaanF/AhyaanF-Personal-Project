HTML Best Practices

Writing Clean and Maintainable HTML

Follow these best practices to write high-quality HTML code:


<!-- ❌ Bad Practice -->
<div class="header">
    <div class="nav"><a href="#">Link</a></div>
</div>
<div class="main">
    <div class="content">
        <img src="pic.jpg">
        <br><br>
        <font color="red">Text</font>
    </div>
</div>

<!-- ✅ Good Practice -->
<!-- Use semantic elements -->
<header>
    <nav>
        <ul>
            <li><a href="#">Link</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <!-- Always include alt text -->
        <img src="pic.jpg" alt="Description">
        
        <!-- Use CSS for styling -->
        <p class="highlight">Text</p>
    </article>
</main>

<!-- Proper indentation and formatting -->
<section class="features">
    <h2>Features</h2>
    <ul>
        <li>Feature 1</li>
        <li>Feature 2</li>
    </ul>
</section>

<!-- Proper form structure -->
<form method="post" action="/submit">
    <fieldset>
        <legend>Contact Information</legend>
        <label for="name">Name:</label>
        <input type="text" id="name" required>
    </fieldset>
</form>

Best Practices:

  • Use Semantic Elements - Choose meaningful tags
  • Proper Indentation - Maintain consistent spacing
  • Descriptive Classes - Use clear, meaningful names
  • Accessibility - Include alt text and ARIA attributes
  • Validate Code - Ensure HTML is valid
  • Avoid Deprecated Tags - Use modern HTML5 elements
  • Consistent Formatting - Follow a style guide
