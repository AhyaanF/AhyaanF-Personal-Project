Semantic HTML

Understanding Semantic Elements

Semantic HTML uses elements that clearly describe their meaning to both the browser and the developer:


<!-- Non-semantic Example -->
<div class="header">
    <div class="nav">
        <div class="nav-item">Home</div>
    </div>
</div>
<div class="main">
    <div class="article">
        <div class="title">My Article</div>
    </div>
</div>
<div class="footer">Copyright 2024</div>

<!-- Semantic Example -->
<header>
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <h1>My Article</h1>
        <section>
            <h2>Article Section</h2>
            <p>Content goes here...</p>
        </section>
        <aside>
            <h3>Related Content</h3>
            <p>Sidebar content...</p>
        </aside>
    </article>
</main>

<footer>
    <p>Copyright 2024</p>
</footer>

Semantic Elements:

  • <header> - Introductory content or navigation
  • <nav> - Navigation links
  • <main> - Main content of the page
  • <article> - Self-contained content
  • <section> - Thematic grouping of content
  • <aside> - Related but separate content
  • <footer> - Footer information
  • <figure> - Self-contained media content
  • <figcaption> - Caption for figure

Practice Exercise

Create a semantic HTML structure for a blog post that includes:

  • A header with navigation
  • Main content area with an article
  • Article sections with headings
  • An aside with related content
  • A footer with copyright information
